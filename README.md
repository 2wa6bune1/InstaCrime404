# 🕯️ 게임 소개

이 프로젝트는 **스마트폰 속 인스타그램 UI를 탐색하며 사건의 진실을 추리하는 미스터리 게임**입니다.

플레이어는 친구의 죽음 이후 남겨진 **스토리, 피드, DM, 부계정, 뉴스 게시물**을 확인하며 사건의 단서를 모읍니다.  
날짜가 하루씩 진행될수록 새로운 게시물과 대화가 열리고, 플레이어는 각 인물의 말과 행동 속에서 모순을 찾아내야 합니다.

이 게임은 단순히 텍스트를 읽는 방식이 아니라, 실제 SNS를 조작하는 것처럼 구성되어 있습니다.

> 스토리를 넘기고, DM을 읽고, 계정을 전환하고, 직접 스토리를 업로드하며  
> 플레이어는 사건의 중심으로 점점 끌려 들어갑니다.

# 🎮 게임 방법

## 1️⃣ 게임 시작

첫 화면에서 **게임 시작** 버튼을 누르면 인트로 영상이 재생됩니다.

인트로가 끝난 뒤, 플레이어는 게임 안에서 사용할 **인스타그램 아이디**를 입력합니다.

```txt
최대 10자
영어, 숫자, _, . 사용 가능
```

입력한 아이디는 게임 속 주인공 계정명으로 표시됩니다.

<br>

## 2️⃣ 핸드폰 열기

게임이 시작되면 방 안에 놓인 핸드폰을 클릭해 인스타그램 화면을 엽니다.

핸드폰 안에서는 다음 기능을 사용할 수 있습니다.

```txt
스토리 확인
피드 게시물 확인
DM 확인
계정 전환
스토리 업로드
```

<br>

## 3️⃣ 날짜별 단서 확인

게임은 날짜 단위로 진행됩니다.

```txt
Day 1
Day 2
Day 3
Day 4
Day 0
```

각 날짜마다 새로운 스토리, 게시물, DM이 등장합니다.

플레이어는 해당 날짜의 주요 단서를 모두 확인해야 다음 단계로 넘어갈 수 있습니다.

<br>

## 4️⃣ 스토리와 DM 탐색

스토리와 DM에는 사건을 풀기 위한 단서가 숨어 있습니다.

특정 스토리나 대화를 확인하면 주인공의 독백이 출력되며, 사건에 대한 정보가 조금씩 정리됩니다.

```txt
스토리 확인
DM 읽기
뉴스 게시물 확인
부계정 단서 확인
독백 확인
```

<br>

## 5️⃣ 4일차 추리

4일차에는 모든 단서를 확인한 뒤, 다음날로 넘어가는 대신 독백이 출력됩니다.

```txt
범인을 알 것 같아. 스토리를 활용해보자
```

이후 플레이어는 직접 스토리를 업로드해 범인을 지목해야 합니다.

<br>

## 6️⃣ 범인 지목

스토리 업로드 화면에서 태그할 계정을 선택합니다.

정답을 고르면 `END` 화면이 출력되고,  
오답을 고르면 `FAILED` 화면이 출력됩니다.

두 경우 모두 **사건의 전말 보기** 버튼을 통해 엔딩 영상을 확인할 수 있습니다.

<br>

## 7️⃣ Day 0

엔딩 영상 이후 게임은 `Day 0`으로 이동합니다.

Day 0에서는 이전 사건과 연결되는 추가 채팅과 영상 단서를 확인할 수 있습니다.

# ✨ 게임 특징

---

## 📱 1. 인스타그램형 UI

게임의 주요 인터페이스는 스마트폰 속 SNS 화면입니다.

```txt
피드
스토리
DM
프로필
계정 전환
스토리 업로드
```

실제 SNS를 탐색하는 느낌을 주기 위해 인스타그램 스타일의 UI를 구현했습니다.

<br>

---

## 📆 2. 날짜 진행 시스템

`DateManager`를 통해 날짜별 데이터가 관리됩니다.

날짜가 바뀔 때마다 다음 요소들이 변경됩니다.

```txt
스토리 목록
피드 게시물
DM 목록
독백 내용
BGM
날짜 전환 영상
```

날짜별 단서가 순차적으로 열리기 때문에, 플레이어는 사건을 단계적으로 추리하게 됩니다.

<br>

---

## 💬 3. DM 추리 시스템

DM에는 인물 간의 관계와 사건 당일의 행적이 담겨 있습니다.

단체 채팅과 개인 채팅을 통해 다음 정보를 파악할 수 있습니다.

```txt
인물들의 위치
사건 당일의 대화
서로 다른 증언
의심스러운 행동
숨겨진 감정
```

읽지 않은 DM은 단서 확인 조건에 포함됩니다.

<br>

---

## 🟣 4. 스토리 단서 시스템

각 인물의 스토리는 날짜마다 다르게 표시됩니다.

스토리를 넘기며 사진, 텍스트, 독백을 확인할 수 있습니다.

특정 스토리는 주인공의 기억이나 사건의 진실과 연결됩니다.

<br>

---

## 👥 5. 계정 전환 시스템

4일차부터는 계정 전환이 가능해집니다.

플레이어는 기존 계정 외에 부계정으로 전환하여 숨겨진 단서를 확인할 수 있습니다.

1~3일차에는 계정 전환 버튼을 눌러도 독백만 출력됩니다.

```txt
자취방이라 인터넷이 잘 안되네
```

<br>

---

## 📸 6. 스토리 업로드 시스템

4일차 후반부에는 플레이어가 직접 스토리를 작성하고 범인을 태그합니다.

스토리 업로드 화면에는 다음 요소가 포함됩니다.

```txt
카메라 화면
지지직거리는 글리치 효과
태그 선택 UI
스토리에 올리기 버튼
END / FAILED 분기
```

<br>

---

## 👻 7. 공포 연출

게임 후반부에는 카메라, 글리치, 피묻은 손 이미지, 엔딩 화면을 활용한 공포 연출이 등장합니다.

```txt
노이즈
화면 지지직 효과
카메라 화면 전환
피묻은 손 연출
END / FAILED 화면
엔딩 영상
```

<br>

---

## 🔁 8. 루프 구조

엔딩 이후 게임은 Day 0으로 이동합니다.

Day 0은 단순한 리셋이 아니라, 이전 사건의 진실과 연결되는 추가 단서 구간입니다.

# 🚀 실행 방법

---

## 1️⃣ 프로젝트 다운로드

```bash
git clone [repository-url]
cd [project-folder]
```

또는 GitHub에서 ZIP 파일로 다운로드한 뒤 압축을 해제합니다.

<br>

---

## 2️⃣ 로컬 서버 실행

이 프로젝트는 이미지, 영상, 사운드, 웹캠을 사용하므로  
`file://` 방식으로 바로 열기보다 로컬 서버에서 실행하는 것을 권장합니다.

### VS Code Live Server 사용

```txt
1. VS Code에서 프로젝트 폴더 열기
2. Live Server 확장 설치
3. index.html 우클릭
4. Open with Live Server 클릭
```

### Python 서버 사용

```bash
python -m http.server 5501
```

브라우저에서 아래 주소로 접속합니다.

```txt
http://127.0.0.1:5501
```

<br>

---

## 3️⃣ 권한 허용

게임 후반부에는 웹캠 연출이 사용됩니다.

브라우저에서 카메라 권한을 요청하면 허용해야 합니다.

```txt
카메라 권한 허용 필요
자동재생 정책 때문에 첫 클릭 이후 사운드 재생 가능
```

<br>

---

## 4️⃣ GitHub Pages 배포

GitHub Pages로 배포할 경우, 다음 사항을 확인해야 합니다.

```txt
index.html이 루트에 있어야 함
assets 폴더 경로가 유지되어야 함
파일명 대소문자가 정확해야 함
영상, 이미지, 사운드 파일이 누락되지 않아야 함
```

특히 GitHub Pages는 파일명 대소문자를 구분합니다.

```txt
movie.mp4
Movie.mp4
MOVIE.mp4
```

위 세 파일명은 서로 다른 파일로 인식됩니다.

# 🔊 사운드 효과

---

## 🎵 배경음악 시스템

게임은 날짜에 따라 다른 BGM을 재생합니다.

```txt
Day 1 ~ Day 4
→ 메인 BGM 재생

Day 0
→ Day 0 전용 BGM 재생

그 외 상태
→ BGM 정지
```

`BgmManager`가 음악의 재생, 반복, 볼륨, 정지를 관리합니다.

<br>

---

## 🎬 영상 사운드

인트로, 날짜 전환, 엔딩 영상에는 영상 자체의 사운드가 포함될 수 있습니다.

```txt

dayTwoVideo.mp4
dayThreeVideo.mp4
dayFourVideo.mp4
endingVideoTest.mp4
movie.mp4
```

영상 재생 중에는 BGM이 멈추거나 화면 연출이 우선됩니다.

<br>

---

## 🔇 브라우저 자동재생 정책

브라우저 정책상 사용자의 첫 클릭 전에는 소리가 자동으로 재생되지 않을 수 있습니다.

게임 시작 버튼 클릭 이후 오디오가 정상 활성화됩니다.
