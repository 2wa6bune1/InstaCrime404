# 🕯️ 게임 소개

이 프로젝트는 **스마트폰 속 인스타그램 UI를 탐색하며 사건의 진실을 추리하는 미스터리 게임**입니다.

플레이어는 친구의 죽음 이후 남겨진 **스토리, 피드, DM, 부계정, 뉴스 게시물**을 확인하며 사건의 단서를 모읍니다.  
날짜가 하루씩 진행될수록 새로운 게시물과 대화가 열리고, 플레이어는 각 인물의 말과 행동 속에서 모순을 찾아내야 합니다.

이 게임은 단순히 텍스트를 읽는 방식이 아니라, 실제 SNS를 조작하는 것처럼 구성되어 있습니다.

> 스토리를 넘기고, DM을 읽고, 직접 스토리를 업로드하며  
> 플레이어는 사건의 중심으로 점점 끌려 들어갑니다.

# 🎮 게임 방법

## 1️⃣ 게임 시작

첫 화면에서 **게임 시작** 버튼을 누르면 인트로 영상이 재생됩니다.

인트로가 끝난 뒤, 플레이어는 게임 안에서 사용할 **인스타그램 아이디**를 입력합니다.

입력한 아이디는 게임 속 주인공 계정명으로 표시됩니다.

<br>

## 2️⃣ 핸드폰 열기

게임이 시작되면 방 안에 놓인 핸드폰을 클릭해 인스타그램 화면을 엽니다.

핸드폰 안에서는 다음 기능을 사용할 수 있습니다.

```txt
스토리 확인
피드 게시물 확인
DM 확인
```

<br>

## 3️⃣ 날짜별 단서 확인

게임은 날짜 단위로 진행됩니다.

```txt
Day 1
Day 2
Day 3
Day 4
```

각 날짜마다 새로운 스토리, 게시물, DM이 등장합니다.

플레이어는 해당 날짜의 주요 단서를 모두 확인해야 다음 단계로 넘어갈 수 있습니다.

<br>

## 4️⃣ 스토리와 DM 탐색

스토리와 DM에는 사건을 풀기 위한 단서가 숨어 있습니다.

특정 스토리나 대화를 확인하면 주인공의 독백이 출력되며, 사건에 대한 정보가 조금씩 정리됩니다.

```txt
스토리 확인
DM 읽기
뉴스 게시물 확인
부계정 단서 확인
독백 확인
```

# 📖 줄거리
🚨 다음 내용은 이 게임의 줄거리로, 스포일러가 포함되어 있습니다.
플레이 후 읽어보시는 것을 추천드립니다.

<br>

## 1️⃣ Day 1

인스타그램 뉴스 계정에 부산에서 20대 여성이 살해당했다는 소식이 올라왔다.
설마 했는데, 피해자는 함께 부산 여행을 갔던 친구들 중 한 명인 서아였다.
어제 부산에서 갑자기 급한 일이 생겼다며 먼저 간다고 했었는데.
그게 마지막이었다니, 믿기지가 않는다.
친구들은 하나둘 서아를 추모하는 스토리를 올리기 시작했다.

그런데 지안이는 이상할 정도로 태연했다.
제일 친한 친구가 살해당한 와중에도 맛집에 가고, 카페에 가고, 평소처럼 일상 스토리를 올리고 있었다.
어떻게 저럴 수가 있는 거지.
내가 예민한 걸까, 아니면 지안이가 이상한 걸까.

그러던 중, 이상한 계정에게서 DM이 왔다.
처음 보는 계정인데, 어쩐지 낯설지가 않았다.
분명 모르는 사람이어야 하는데, 이상하게 익숙한 느낌이 들었다.

<br>

## 2️⃣ Day 2

서아가 교살당했다는 이야기를 들었다.
얼마나 아팠을까. 대체 어떤 사람이 그런 짓을 한 걸까.
빨리 진실이 밝혀지길 매일 기도하고 있다.

그런데 의문의 계정이 피가 묻은 듯한 이어폰 사진을 스토리에 올렸다.
대체 무슨 계정이지.
왜 이런 사진을 올리는 거지.
그리고 나는 왜 이 계정을 팔로우하고 있는 걸까.

서준이는 지안이와 수진이의 스토리를 보고 많이 화가 난 듯했다.
같이 놀러 갔던 친구가 죽은 지 이틀밖에 되지 않았는데, 누군가 아무렇지 않게 일상을 올리는 모습이 이해되지 않는 것도 당연했다.
그래도 친한 친구 스토리로 대놓고 저격이라니.
이러다 친구들 사이가 완전히 틀어지는 건 아닐까 걱정이 됐다.

하은이에게서도 답장이 왔다.
올해 들어 자꾸 기절하는 일이 생기는데, 서아가 죽고 난 뒤로 더 심해졌다고 했다.
나도 이상하게 기억이 끊기는 순간들이 자꾸 생긴다.
종교의 도움을 받아야 하나.
병원을 가야 하나.
머릿속이 복잡하다.

그런데 문득, 이상한 생각이 들었다.
서아는 교살당했다.
그리고 그 계정은 피 묻은 이어폰을 올렸다.
설마.
이어폰으로도 교살이 가능한 걸까.

<br>

## 3️⃣ Day 3

여행을 갔던 친구들의 갈등이 심해지고 있다…. 
내일이 서아 장례식인데 서준이가 올린 스토리 때문에 다들 날이 서있네…

요즘 내 기절 증삼이 점점 심해지는 것 같아서 하은이에게 상담을 받았는데 악령이라니 뭐라니….
나 진짜 이상해지는 걸까...

이상한 계정에는 약사진이 올라왔다.
평소 같았으면 신경도 안썼겠지만 나랑 상황이 비슷한걸까..

그리고 지안이한테 내가 맛집을 알려줬다고? 
기록은 남아 있는데 왜 기억이 전혀 나지 않는 거지?

## 4️⃣ Day 4

하은이한테는 비밀이지만…. 정신병원에 다녀왔다.
증상이 계속해서 심해지고 있는 것 같다.

오늘은 서아의 장례식… 
큰일은 없었지만 다들 서로에게 여전히 의심을 품고 있는 것 같다.

그렇지만… 이제는 범인을 알 것 같아

## 🧩 주요 파일 역할

### `index.html`

프로젝트의 진입점입니다.

브라우저에서 실행될 때 필요한 p5.js 라이브러리와 각 JavaScript 파일을 불러옵니다.

```txt
index.html
→ p5.js 로드
→ 프로젝트 JS 파일 로드
→ sketch.js 실행
```

<br>

---

### `sketch.js`

게임 전체의 중심이 되는 메인 파일입니다.

p5.js의 `preload()`, `setup()`, `draw()`, `mousePressed()`, `keyPressed()` 흐름을 관리합니다.

주요 역할은 다음과 같습니다.

```txt
전체 게임 상태 관리
시작 화면 출력
아이디 입력 처리
영상 재생
```

<br>

---

### `dateManager.js`

날짜별 게임 데이터를 관리하는 파일입니다.

날짜별로 달라지는 스토리, 피드, DM, 독백 내용을 로드합니다.

```txt
현재 날짜 관리
날짜별 스토리 데이터 생성
날짜별 피드 게시물 생성
날짜별 DM 채팅 데이터 생성
날짜별 시작 / 종료 독백 제공
날짜별 기능 작동 여부 관리
```

`dateManager.loadDailyData()`는 날짜별 데이터를 생성한 뒤, 인스타그램 UI 쪽으로 전달합니다.

```txt
DateManager
→ dailyStories 생성
→ dailyPosts 생성
→ dailyChats 생성
→ InstagramUI.loadData()
→ InstagramDM.loadChatData()
```

<br>

---

### `phoneUI.js`

화면 속 핸드폰 자체를 담당하는 파일입니다.

핸드폰의 위치, 크기, 확대, 축소, 뒤집힘, 화면 클리핑 등을 관리합니다.

```txt
핸드폰 외형 출력
핸드폰 확대 / 축소 애니메이션
핸드폰 뒤집힘 연출
인스타그램 화면 표시
폰 내부 마우스 좌표 계산
홈 버튼 처리
스크롤 전달
```

`PhoneUI` 내부에는 `InstagramUI`가 들어 있습니다.

```txt
PhoneUI
└── InstagramUI
```

즉, 플레이어가 보는 인스타그램 화면은 실제로는 핸드폰 안쪽에 그려지는 UI입니다.

<br>

---

### `instagramUI.js`

인스타그램 앱 전체 화면을 담당하는 파일입니다.

피드, 스토리, 프로필, DM, 스토리 업로드 화면으로 이동하는 큰 구조를 관리합니다.

```txt
현재 인스타그램 화면 상태 관리
피드 출력
스토리 아이콘 출력
스토리 재생
프로필 화면 출력
하단 네비게이션 처리
DM 화면 연결
스토리 업로드 화면 연결
계정 상태 관리
```

주요 화면 상태는 다음과 같습니다.

```txt
feed
story
profile
dmList
chatRoom
storyUpload
```

```txt
계정 전환 시트 출력
```

<br>

---

### `storyUploader.js`

4일차 후반부의 스토리 업로드와 범인 지목 시스템을 담당합니다.

```txt
스토리 카메라 화면
글리치 효과
태그 선택 UI
스토리에 올리기 버튼
```

정답을 선택하면 카메라 연출과 손자국 연출 이후 `END` 화면이 출력됩니다.

오답을 선택하면 `FAILED` 화면이 출력됩니다.

두 경우 모두 `사건의 전말 보기` 버튼을 누르면 엔딩 영상으로 이동합니다.

<br>

### `monologue.js`

```txt
DateManager
Story
InstagramDM
Room
StoryUploader
→ MonologueSystem.start()
```

# 🚀 실행 방법

---

## 1️⃣ 프로젝트 다운로드

```bash
git clone [repository-url]
cd [project-folder]
```

또는 GitHub에서 ZIP 파일로 다운로드한 뒤 압축을 해제합니다.

<br>

---

## 2️⃣ 로컬 서버 실행

이 프로젝트는 이미지, 영상, 사운드, 웹캠을 사용하므로  
`file://` 방식으로 바로 열기보다 로컬 서버에서 실행하는 것을 권장합니다.

### VS Code Live Server 사용

```txt
1. VS Code에서 프로젝트 폴더 열기
2. Live Server 확장 설치
3. index.html 우클릭
4. Open with Live Server 클릭
```

### Python 서버 사용

```bash
python -m http.server 5501
```

브라우저에서 아래 주소로 접속합니다.

```txt
http://127.0.0.1:5501
```
